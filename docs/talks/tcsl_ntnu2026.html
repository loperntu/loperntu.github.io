詞意教學的導航：Chinese Wordnet CWN

如何利用中文詞彙網路豐富的語意關係，協助學生精準掌握詞義辨析？

📖 什麼是 Chinese Wordnet？

Chinese Wordnet（中文詞彙網路）https://lopentu.github.io/CwnWeb/ 是以 Princeton WordNet 為藍本，專為（當代）漢語建置的大型詞彙語意知識庫。它不只是一本「電子辭典」，更是一張巨大的語意地圖 —— 每個詞意都是地圖上的一個節點，節點之間以各種語意關係（同義、反義、上下位、部分-整體⋯⋯）相互連結。

對華語教師而言，CWN 就像是一套 詞彙教學的 GPS：它能告訴你一個詞「在哪裡」（語意位置）、「附近有什麼」（相關詞彙）、以及「怎麼去」（從已知詞彙通往未知詞彙的路徑）。

詞條（字面形式）

詞意（一詞多義）
群

🌳 CWN 的資料結構：以「打」為例

讓我們用華語學習者最害怕的動詞之一 ——「打」——來看看 CWN 如何組織詞彙資訊：

Lemma 詞條 打 (ID: 06613)
Sense 01 用手或器具撞擊。「打人」「打鼓」
↑ 上位詞 撞擊
→ 同義 擊、敲
Sense 02 從事某種活動。「打球」「打牌」
↑ 上位詞 從事
Sense 03 以電話等方式聯繫。「打電話」
Sense 04 編織、製作。「打毛衣」「打結」
⋯⋯（CWN 收錄「打」超過 120 個義項）
那「吧」呢？

🔗 語意關係：教學的核心武器

CWN 裡的語意關係不只是「語言學知識」，它們就是教學策略的骨架：

語意關係教學應用例子
上下位
Hypernymy		 從「大概念」出發，引導學生收斂到具體詞彙，或由具體詞彙歸納上位概念 交通工具 → 車、船、飛機
水果 → 蘋果、香蕉、芒果
同義關係
Synonymy		 辨析近義詞，強調語用/語域差異 看 vs. 瞧 vs. 觀看 vs. 注視
（正式程度遞增）
反義關係
Antonymy		 成對教學，利用對比加深記憶 冷↔熱、開心↔難過、進↔出
部分-整體
Meronymy		 建構「場景詞彙包」，情境教學 身體 → 頭、手、腳⋯
💡 教學洞察

傳統教材是線性排列的詞彙表，學生只能「前後」記憶。CWN 提供的語意網路讓學生可以「上下左右」地拓展詞彙，這正是母語者心理詞彙（mental lexicon）的組織方式。教師可以利用 CWN 設計以「概念」為核心而非以「課文」為核心的詞彙教學活動。

🛠 如何取用 CWN 資料？

CWN 提供了多種取用方式，適合不同使用情境：

🌐 網頁查詢

CwnWeb 是一個完全靜態的網頁介面，任何人都可以直接搜尋、瀏覽 CWN 的詞條與語意關係。適合教師備課時快速查詢。

🐍 Python 程式套件

透過 CwnGraph 套件，可以用 Python 程式化地存取整個詞彙網路，適合進階應用：自動化教材生成、研究分析、或開發教學 App。

# Python 快速上手
from CwnGraph import CwnImage
cwn = CwnImage.latest()
# 查詢「打」的所有詞意
senses = cwn.find_senses("打")
for s in senses:
  print(s.definition)

🎯 教學活動設計：「語意星空」

利用 CWN 的上下位關係設計「語意星空」活動——讓學生從一個核心概念出發，像星座一樣向外擴展相關詞彙：

情緒
（核心概念）
→
正面
開心、興奮、滿足
→
負面
難過、生氣、焦慮
→
開心 → 高興/愉快/快樂
（同義辨析）
⟷
難過 → 傷心/悲傷/哀痛
（正式度差異）

學生先從上位概念理解整體分類，再深入同義詞辨析，最後反義配對。每一步都可以有 CWN 的語意關係作為「鷹架」（scaffolding），幫助學生理解詞意。甚至將語意場（或結合 FrameNet 的概念融入，讓學生可以在使用脈絡中學習。

捕捉真實語言的律動：社群媒體語料庫 Corpus

傳統教材與當代語言之間的落差，正是語料庫可以彌補的。

🌍 為什麼需要社群媒體語料庫？

華語教學有一個長期存在的痛點：教材裡教的，跟真實世界說的，往往不一樣。

📚 教材的語言

「你好，請問這件衣服多少錢？」
「三百五十元。」
「好的，謝謝你。」

（標準、規範、但有時不自然）

💬 社群的語言

「這件也太好看了吧！求連結🙏」
「+1 想知道在哪買的」
「已私你～」

（真實、動態、充滿創意）

社群媒體語料庫讓我們能夠系統性地觀察當代漢語的「活」語言——包括新詞、流行語、語碼混用、非典型句構等等，這些都是進階學習者在真實溝通中必須面對的。

🔍 兩岸漢語的動態變異

此外，透過語料庫比較，我們也可以觀察到世界漢語在詞彙、語法、甚至語用上的有趣差異：

面向台灣華語中國普通話
詞彙 軟體、網路、資訊、捷運 軟件、網絡/互聯網、信息、地鐵
新詞 台味、佛系、躺平 內卷、擺爛、整頓職場
語氣詞 齁、吼、蛤、欸 呦、唄
程度表達 超、有夠、爆 巨、絕了、YYDS

這些差異不只是「詞彙表」的問題。它們反映了不同社會文化脈絡下語言使用者的認知框架與互動策略。
【我們走吧】vs 【我們走唄】

💬 語料庫中的真實語言樣貌

以下是從社群媒體語料庫擷取的真實語言使用例：

「這家店的牛肉麵也太好吃了吧！排了一個小時完全值得」
── PTT 美食版
「不是，你認真的嗎？這個價格也太扯了」
── Dcard 閒聊版
「欸我跟你說，昨天那個什麼什麼的，你有看嗎？」
── Facebook 留言

注意這些例子中的「也太⋯了吧」、「不是，」（感嘆標記）、「我跟你說」（話語標記）——這些都是教材裡不太會教，但在真實對話中頻繁出現的構式（construction）。

📐 從 Notional Syllabus 到構式教學

D.A. Wilkins（1972）提出的 Notional Syllabus（意念大綱）主張：語言教學應該以「表達什麼意義」（notions/functions）而非「教什麼語法」來組織課程。這個理念與當代構式語法（Construction Grammar）不謀而合：

意念
Notion
「表達不滿」
＋
功能
Function
「抱怨、責備」
→
構式
Construction
「X 什麼 X」

構式是「形式-意義」的配對體（form-meaning pairing）。社群語料庫讓我們可以大量觀察這些構式在真實使用中的分佈（distribution）和語用制約（pragmatic constraints），進而設計出更貼近真實的教學活動。

🧩 華語教學中的高頻構式

構式模板意念/功能例句情緒傾向
X 什麼 X 否定、不耐煩 「吃什麼吃，你沒看到我在忙嗎？」 😤 負面
還在那邊 XX 責備、嫌棄 「都幾點了還在那邊睡覺！」 😒 負面
也太 X(X) 了吧 驚訝（超出預期） 「這也太好吃了吧！」 😲 中性偏正
V 就 V，誰怕誰 挑戰、決心 「走就走，誰怕誰！」 💪 強勢
不 V 不知道，一 V 嚇一跳 意外發現 「不查不知道，一查嚇一跳！」 😮 驚訝

🖌 從偵錯到書法文化

漢字偵錯不只是「找出錯字」，它也是通往漢字文化的窗口。當 AI 解釋「已」和「己」的差別時，可以同時呈現：

  • 字形分析：「已」的末筆出頭，「己」的末筆不出頭——在書法中，這個微小差異可以用「收」與「放」的概念來理解
  • 字源故事：從甲骨文、金文的演變看漢字如何「長」成現在的樣子
  • 形近字系統：已、己、巳——三個字只差一點，AI 可以自動生成比較表和記憶口訣
  • 書法體驗：搭配數位書法工具，讓學生感受「筆意」——漢字不只是符號，更是書寫的藝術
